# Elgreensyde v3.2: Operational Integration Walkthrough

This document outlines how the four core modules (Monitoring, Tasks, Maintenance, and Batches) work together as a single automated loop to ensure farm safety and efficiency.

---

## 🔄 The Integrated Loop

### 1. Scouting & Diagnosis (Monitoring)
The loop begins in **Monitoring.jsx**. Instead of generic text notes, the farm team now selects categories and symptoms from a structured tree.
- **Action:** Select a Bed -> Choose Category (e.g., Fungal) -> Choose Symptom (e.g., Downy Mildew).
- **Trigger:** Saving the session inserts a row into `flagged_issues` with `is_active_threat = true`.

### 2. Auto-Task Generation (The "Zap" Engine)
The system monitors these flags. When a critical threat is identified, it auto-generates a task with high priority.
- **Aphids** -> Generates: *"IPM: Cull infested Nasturtium trap crops"*
- **Downy Mildew** -> Generates: *"CRITICAL: Apply Copper Fungicide Drench"*
- **Fusarium** -> Generates: *"BIOSECURITY: Quarantine and Test"*

### 3. Action & Inventory (Maintenance)
The technician sees the task and goes to **Maintenance.jsx** to perform the treatment.
- **Required Input:** The user MUST select the **Consumable SKU** (Chemical/Fertilizer) and the **Amount Used**.
- **Automation:** Saving the maintenance log triggers the `decrement_inventory` RPC.
- **Result:** Inventory stock levels drop automatically. No manual stock counting is required for used chemicals.

### 4. Safety Locking (Batches & Plots)
The **Batches.jsx** monitor acts as the final safety gate.
- **Visual Alert:** Any bed with an active threat displays a pulsing red border.
- **Quarantine Lock:** The system checks for recent maintenance. If a chemical with a `withholding_period` was applied, the **Log Harvest** button is disabled.
- **Feedback:** A countdown label Appears: *"Quarantined until [Date]"*.

## 🛠 Database & Technical Connections

| Source Module | Target Table | Triggered Effect |
| :--- | :--- | :--- |
| **Monitoring** | `flagged_issues` | Activates red borders in Batches UI |
| **Monitoring** | `tasks` | Auto-populates the daily to-do list |
| **Maintenance** | `inputs_inventory` | Live stock deduction (RPC) |
| **Maintenance** | `maintenance_logs` | Calculates harvest safety window |

## 📝 Compliance Note
This integration ensures that no crop is ever harvested while under a chemical withholding period, fulfilling the farm's **Food Safety Compliance** requirements for v3.2.
