System Overview

Elgreensyde is a solo-operated herbs and edible flowers farm in Cagayan de Oro, Philippines. This document defines the complete data architecture, business logic, and UI specifications for a custom farm management web application. The system is designed to handle all operations from propagation through to point of sale.

Current farm scope:

  • Potted herbs — primary product line, sold per unit
  • Basil and Parsley — grown in raised beds/ground plots, sold per gram
  • Sales channels: walk-in buyers and online orders with delivery

Module 1: Cultivation — The Two Pipelines

The cultivation module is the backbone of the system. Because Elgreensyde grows herbs in two fundamentally different formats, the database must treat them differently — but the dashboard must present them together seamlessly.

Transplant & Assign Workflow

When a tray reaches Ready for Transplant status, a Transplant & Assign button triggers a modal with:

  • Source: Auto-filled with Tray ID and Crop Type
  • Destination Dropdown: Filtered to only show plots with Cleared or Resting status
  • Transplant Date: Auto-filled to today, editable for late entries

On Save, the system executes the following chain:

  • Tray ID is marked Archived / Completed
  • Physical plot changes status from Cleared to Active
  • Crop type and original sowing date transfer to the plot's active cycle

Unified Dashboard View

The daily dashboard presents both pipelines side by side:

  • Left Column (Plot Monitor): All Basil and Parsley plots. Highlights plots not harvested in 14+ days.
  • Right Column (Batch Monitor): All potted herb batches in Nursery stage, sorted by start date.

True Margin Per Batch

The system calculates batch-level profitability using Input Cost Tracking:

  • Gross Revenue: Market-Ready Qty × Retail Price (e.g., 47 pots × ₱150 = ₱7,050)
  • Direct Costs: Input cost logged at batch creation (e.g., ₱1,000 for 50 pots)
  • Net Profit: Gross Revenue minus Direct Costs (e.g., ₱6,050)
  • Wastage Value: Dead pots × Retail Price (e.g., 3 × ₱150 = ₱450 — shown in Efficiency panel only)

Over time, comparing True Margin across herb varieties identifies which crops deliver the best return on input cost and which require a process change.

Module 8: Planting Calendar

The Planting Calendar shifts the system from reactive (recording what happened) to predictive (planning what needs to happen). It is activated once baseline nursery-day data has been collected from actual batch records.

How It Works

  • Set a weekly output target per SKU (e.g., 20 pots of Peppermint, 2kg of Sweet Basil)
  • The system reads the average nursery days from historical batch records for that crop
  • It back-calculates the required sow date to meet the target harvest window
  • An alert fires if you are approaching the sow date without an active tray for that crop

Key Data Flow Chains

Harvest → Inventory:

  • Log Harvest on PLT-BSL-01 (500g) → harvest_logs entry created → inventory SKU-BSL-BLK += 500

Batch Transfer → Inventory:

  • Click Move to Inventory on BCH-MNT-04 (47 pots) → batch status = Completed → inventory SKU-MNT-POT += 47

POS Fulfillment → Ledger:

  • Fulfilled POS transaction → inventory deducted → financial_ledger Revenue entry created

Maintenance Log → Task Alert:

  • Last fertilizer log date checked daily → if elapsed days > crop threshold → task auto-created
